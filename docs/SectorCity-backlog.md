# «TerraForm Drive» — backlog разработки

> Третий документ комплекта (после ТЗ и технической архитектуры). Здесь фазы разбиты на мелкие задачи-«кусочки» — каждую можно завести как отдельную ветку/PR/тикет.
>
> **Допущения:** ~8 недель, Windows-first. **Стек зафиксирован:** Rust + Tauri v2 (`jwalk`, `rusqlite`) · Svelte + TS + Vite (`three`, `d3-hierarchy`, `nanostores`, `@tweenjs/tween.js`, `MapControls`) · машина режимов на discriminated union · ручная проекция · highlight-mesh · CPU raycast.

## Как работать с этим списком

- **Один кусочек = одна ветка/PR.** Маленький, проверяемый, мёржится отдельно.
- **Не начинать новую фазу, пока не закрыт её DoD.** MVP (фаза 1) — страховочный пол: даже если дальше не успеть, это сдаваемый продукт.
- **Оценки:** `S` ≈ ≤0.5 дня · `M` ≈ 1 день · `L` ≈ 2+ дня. Это ориентир, не обязательство.
- Порядок внутри фазы — это и порядок зависимостей: верхнее нужно нижнему.
